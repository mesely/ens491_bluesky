# BlueSky Turkish Political Feed Analysis Pipeline

**ENS491 Graduation Project — Sabancı University**

An end-to-end data pipeline that collects, analyses, and visualises Turkish political discourse on the Bluesky social network. The pipeline verifies politician accounts via the AT Protocol API, fetches their posts, extracts political keywords, runs Turkish-language sentiment and hate speech analysis, and builds an interaction network graph.

---

## Project Architecture

```
PHASE 1 — Account Verification & Data Collection
  └─ 1a. Verify bsky_handle values via AT Protocol API
  └─ 1b. Fetch all posts for each verified account (pagination)
  └─ 1c. Extract political keywords (TF-IDF + curated seed list)

PHASE 2 — Bluesky-wide Weekly Search
  └─ 2a. Search Bluesky for top-50 keywords in the last 7 days
  └─ 2b. Compute post distribution statistics

PHASE 3 — Sentiment Analysis (TurkishBERTweet)
  └─ 3a. Preprocess text (TurkishBERTweet Preprocessor)
  └─ 3b. Sentiment: negative / neutral / positive (Lora-SA)
  └─ 3c. Hate speech detection: Yes / No (Lora-HS)

PHASE 4 — Network Analysis
  └─ 4a. Build reply + quote interaction graph
  └─ 4b. Compute centrality metrics and community structure
  └─ 4c. Interactive network visualisation (PyVis)

PHASE 5 — Visualizations & Report
  └─ 5a. 9 figures (PNG + interactive HTML)
  └─ 5b. Summary statistics
```

## Quick Start

### 1. Clone and set up environment

```bash
git clone <repo-url>
cd ens491_bluesky

python -m venv venv
source venv/bin/activate          # Windows: venv\Scripts\activate

pip install atproto pandas openpyxl requests tqdm python-dotenv \
            scikit-learn networkx pyvis matplotlib seaborn plotly \
            wordcloud jupyter ipykernel

# For GPU-accelerated sentiment analysis:
pip install torch --index-url https://download.pytorch.org/whl/cu118
pip install transformers peft
```

### 2. Configure credentials

```bash
cp .env.example .env
# Edit .env and fill in your Bluesky handle and App Password
# Get an App Password from: Settings → App Passwords → Add App Password
```

### 3. Clone TurkishBERTweet (required for Phase 3)

```bash
git clone https://github.com/ViralLab/TurkishBERTweet.git
```

### 4. Run the full pipeline

**Option A — Jupyter Notebook (recommended):**
```bash
jupyter notebook analysis.ipynb
```
Run all cells in order. Each section shows a sample of its output.

**Option B — Individual scripts:**
```bash
python src/01_verify_accounts.py
python src/02_fetch_posts.py
python src/03_keyword_extraction.py
python src/04_weekly_search.py
python src/05_sentiment_analysis.py
python src/06_network_analysis.py
python src/07_visualizations.py
```

| File | Description |

| `outputs/network_metrics.json` | Centrality rankings, community detection, intra/inter-party ratios |

| `VRLLab/TurkishBERTweet-Lora-SA` | Sentiment Analysis | negative · neutral · positive |

GPU is used automatically if available (`torch.cuda.is_available()`).

## Research Questions Answered by the Pipeline

1. **Sentiment distribution per party** — which party posts most positively/negatively?
2. **Cross-party negativity** — which party speaks most negatively about others?
3. **Hate speech rates** — which parties have the highest hate speech prevalence?
4. **Overall sentiment landscape** — positive vs neutral vs negative across all political discourse.
5. **Milletvekili vs non-milletvekili** — do elected officials tweet differently than other political actors?

## AT Protocol Notes

- **Public API** (`public.api.bsky.app`) — no login required, ~50–100 req/min limit.
- **DID** (Decentralized Identifier) is stable; handles can change, DIDs cannot.
- **Custom domain handles** (e.g. `omercelik.com`) are resolved transparently by the API.
- All scripts respect rate limits with `time.sleep(0.3)` between requests and back-off on `429`.
